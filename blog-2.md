
## How do `Pick` and `Omit` utility types prevent code duplication while creating specialized "slices" of a master interface? Discuss how this keeps your code **DRY** (Don't Repeat Yourself).

### Generics

জেনেরিকস হলো টাইপস্ক্রিপ্টের অন্যতম শক্তিশালী একটি ফিচার, কারণ এটি ডেভেলপারদের কঠোর টাইপ সেফটি বজায় রেখেও পুনঃব্যবহারযোগ্য এবং নমনীয় কোড তৈরি করতে সাহায্য করে। বিভিন্ন ডেটা টাইপের জন্য আলাদা ফাংশন বা কম্পোনেন্ট লেখার পরিবর্তে, জেনেরিকস আপনাকে এমন একটিমাত্র কোড লিখতে দেয় যা টাইপের তথ্য না হারিয়েই অনেক ধরনের টাইপের সাথে কাজ করতে পারে। এর ফলে অ্যাপ্লিকেশন রক্ষণাবেক্ষণ করা সহজ হয়, কোডের পুনরাবৃত্তি কমে এবং স্কেলেবিলিটি উন্নত হয়। উদাহরণস্বরূপ, একটি জেনেরিক ফাংশন ডেভেলপমেন্টের সময় সঠিক টাইপ চেকিং বজায় রেখেই স্ট্রিং, সংখ্যা, অবজেক্ট বা অ্যারে গ্রহণ করতে পারে।

```tsx
function getFirstElement<T>(array: T[]): T {
    return array[0];
}

const numbers = getFirstElement<number>([10, 20, 30]);
const names = getFirstElement<string>(["Alice", "Bob", "Charlie"]);

console.log(numbers); // 10
console.log(names);   // Alice
```

 উপরের কোড উদাহরণে, <T> একটি জেনেরিক টাইপ প্লেসহোল্ডারকে বোঝায়। getFirstElement ফাংশনটি T[] টাইপের একটি অ্যারে গ্রহণ করে, যার মানে এটি যেকোনো ধরনের ডেটা ধারণকারী অ্যারের সাথে কাজ করতে পারে। এর রিটার্ন টাইপও T, যা নিশ্চিত করে যে ফেরত আসা মানটি অ্যারের উপাদানগুলোর টাইপের সাথে মেলে। যখন getFirstElement<number>([10, 20, 30]) কল করা হয়, TypeScript T-কে number দিয়ে প্রতিস্থাপন করে। একইভাবে, যখন getFirstElement<string>(["Alice", "Bob", "Charlie"]) ব্যবহার করা হয়, তখন T হয়ে যায় string। এটি একই ফাংশনকে বিভিন্ন ডেটা স্ট্রাকচারের জন্য সম্পূর্ণ টাইপ-সেফ থাকার পাশাপাশি পুনঃব্যবহারযোগ্য থাকতে সাহায্য করে।
