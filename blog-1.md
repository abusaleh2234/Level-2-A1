## Type Safety in TypeScript: any, unknown, and Type Narrowing
### any Type 

জাভাস্ক্রিপ্টে ডিফল্ট টাইপ হলো “any”, যার অর্থ হলো একটি ভ্যালু যেকোনো ধরনের হতে পারে। তবে, টাইপস্ক্রিপ্ট ব্যবহারের মূল উদ্দেশ্য হলো “any” টাইপকে এড়িয়ে চলা, কারণ এটি টাইপস্ক্রিপ্টের প্রাথমিক উদ্দেশ্যের পরিপন্থী। টাইপস্ক্রিপ্টের প্রাথমিক উদ্দেশ্য হলো আরও কঠোর টাইপ চেকিং প্রদান করা এবং জাভাস্ক্রিপ্টের ডিফল্ট আচরণের  অস্পষ্টতা দূর করা।


```tsx
let city: any = "Dhaka"
// can assign object 
city = {
    name: "Barishal",
    district: 6
}
// can assign Number
city = 123;
// can assign array
city  = ["Dhaka", "Rajshahi", "Khulna"]
```

এখানে, “city” ভেরিয়েবলটি “any” টাইপের হতে পারে। এর মানে হলো, আপনি “city” ভেরিয়েবলে যেকোনো ধরনের ভ্যালু স্টোর করতে পারেন এবং TypeScript কোনো আপত্তি জানাবে না, এমনকি যদি আপনি এতে অ্যাসাইন করা ভ্যালুর টাইপ পরিবর্তনও করেন। উদাহরণস্বরূপ, আপনি এখানে দেখানো অনুযায়ী, “city”-এ একটি স্ট্রিং-এর পরিবর্তে একটি সংখ্যা বা একটি অ্যারে অ্যাসাইন করতে পারেন। আপনি দেখতে পাচ্ছেন যে “city”-এ একটি সংখ্যা অ্যাসাইন করা হয়েছে এবং TypeScript কোনো অভিযোগ করছে না। “any” টাইপ আপনাকে একটি ভেরিয়েবলে যেকোনো ধরনের ভ্যালু অ্যাসাইন করার সুযোগ দেয়, যা সমস্যার কারণ হতে পারে। অ্যাসাইন করা ভ্যালু যাই হোক না কেন, TypeScript কোনো অভিযোগ করবে না। একটি ভেরিয়েবলে “any” অ্যাসাইন করা অনেকটা জাভাস্ক্রিপ্টকে স্বয়ংক্রিয়ভাবে ভ্যালুর টাইপ নির্ধারণ করতে দেওয়ার মতো। TypeScript-এ আমরা এই আচরণটি এড়াতে চাই, কারণ এটি জাভাস্ক্রিপ্টের ডিফল্ট আচরণ। জাভাস্ক্রিপ্ট অ্যাসাইন করা ভ্যালুর উপর ভিত্তি করে ভেরিয়েবলের টাইপ অনুমান করার চেষ্টা করে। এই ক্ষেত্রে, জাভাস্ক্রিপ্ট অনুমান করবে যে “city” হলো “string” টাইপের, কিন্তু যেই মুহূর্তে একটি সংখ্যা অ্যাসাইন করা হবে, জাভাস্ক্রিপ্ট অনুমান করবে যে এটি একটি সংখ্যা। এখানেই TypeScript-এর ভূমিকা শুরু হয়। টাইপস্ক্রিপ্টের উদ্দেশ্য হলো আমাদের ভ্যালুগুলোর উপর স্ট্রিক্ট টাইপ প্রয়োগ করা, এবং “any” টাইপটি শেষ উপায় হিসেবে খুব কম ব্যবহার করা উচিত। টাইপস্ক্রিপ্টে “any” টাইপের অস্তিত্ব সম্পর্কে অবগত থাকা ভালো হলেও, এটি ঘন ঘন ব্যবহার করা উচিত নয়।


### unknown Type

টাইপস্ক্রিপ্টে “unknown” টাইপটি “any” টাইপের সাথে বেশ সাদৃশ্যপূর্ণ, তবে কিছু সূক্ষ্ম পার্থক্য রয়েছে যা নিয়ে আমাদের আলোচনা করা উচিত।

```tsx
function textLengthWithAnyType (text: any) {
    return text.length;
}
function textLengthWithunknownType (text: unknown) {
    return text.length;
}
```

“textLengthWithAnyType” এবং “textLengthWithunknownType”। “textLengthWithAnyType” ফাংশনটির “text” নামের একটি প্যারামিটার আছে যার টাইপ 'any', অপরদিকে “textLengthWithunknownType” ফাংশনটির প্যারামিটারটির টাইপ 'unknown'। 

আপনি চিত্রে দেখতে পাবেন যে “textLengthWithunknownType”-এর রিটার্ন স্টেটমেন্টটি একটি লাল আন্ডারলাইন দিয়ে চিহ্নিত করা আছে। এটি নির্দেশ করে যে টাইপস্ক্রিপ্ট একটি ত্রুটি চিহ্নিত করছে। ত্রুটির বার্তাটি থেকে বোঝা যায় যে 'text'-এর টাইপ 'unknown'। টাইপস্ক্রিপ্ট মূলত বলছে যে এই ফাংশনের 'text' প্যারামিটারের টাইপ নির্ধারণ করার জন্য তার কাছে যথেষ্ট তথ্য নেই। তাই, এটি 'text'-এর দৈর্ঘ্য গণনা করতে বা প্রত্যাশিত আউটপুট প্রদান করতে পারবে কিনা, তার নিশ্চয়তা দিতে পারছে না।

এটি সঠিক, কারণ যখন এই ফাংশনটিকে আর্গুমেন্ট হিসেবে একটি স্ট্রিং দিয়ে কল করা হয়, তখন এটি প্রত্যাশিত আউটপুট তৈরি করে। তবে, যদি আর্গুমেন্ট হিসেবে একটি সংখ্যা ব্যবহার করা হয়, তাহলে ফাংশনটি প্রত্যাশিতভাবে কাজ করে না, যে কারণে টাইপস্ক্রিপ্ট একটি এরর দেখায়।
